---
description: Принять зачёт по пройденной теме
---

Передай управление агенту `examiner`. Он проведёт зачёт по теме текущей задачи.

## После окончания зачёта (автоматически)

Выполни все шаги последовательно:

### Шаг 1: Сохрани фидбек examiner
Создай `.claude/notes/claude/<тема>-feedback.md` со структурой:
```
# Фидбек: <Название задачи/темы>

**Дата**: YYYY-MM-DD
**Источник**: <LeetCode #N / Kaggle / ...>
**Оценка**: X/10

## Сильные стороны
<из фидбека examiner>

## Зоны роста
<из фидбека examiner — конкретные подтемы>

## Рекомендованные задачи для закрепления
<из рекомендаций examiner>
```

Затем создай шаблон для заметки пользователя `.claude/notes/my_notes/<тема>.md`:
```
# <Название задачи/темы>

**Дата**: YYYY-MM-DD
**Источник**: <LeetCode #N / Kaggle / ...>

## Ключевые идеи
<!-- Что я понял? Запиши своими словами -->

## Инсайты
<!-- Что было неочевидно и стало понятно в процессе? -->

## Что хочу запомнить
<!-- Главное, что вынес из задачи -->
```
Скажи пользователю: "Заполни заметку в `.claude/notes/my_notes/<тема>.md` и запусти `/export-obsidian`"

### Шаг 2: Архивируй задачу
Перенеси содержимое `.claude/tasks/current/` в `.claude/tasks/archive/<YYYY-MM-DD>-<название>/`
Создай пустой шаблон в `.claude/tasks/current/task.md` (скопируй из архива, очисти содержимое)

### Шаг 3: Экспорт в Obsidian
Экспорт происходит вручную после того, как пользователь заполнит свою заметку в `my_notes/`. Напомни: "/export-obsidian когда будешь готов".

### Шаг 4: Предложи следующий шаг
После всех шагов напиши:
- Что изучать дальше, исходя из слабых мест
- Конкретные задачи или темы для `/new-task`
