# Claude Coding Mentor

Набор агентов и команд для Claude Code, превращающий его в персонального ментора по алгоритмам и Data Science. Ментор не решает задачи за тебя — он проводит тебя через процесс понимания и обучения шаг за шагом.

---

## Концепция

Обычный Claude даёт готовые ответы. Этот проект надстраивает над ним **педагогический слой**: сократический метод, поэтапное раскрытие информации, проверку понимания и экспорт знаний в Obsidian.

Менторский режим активируется только через слэш-команды — в остальное время Claude ведёт себя как обычно.

## Агенты

### `algo-teacher` — Учитель теории
Объясняет концепции через аналогии → механику → формализм → применение → подводные камни. После каждого объяснения задаёт 3 контрольных вопроса. Используется командой `/explain`.

### `code-coach` — Тренер по коду
Подключается когда алгоритм уже понят и нужно его реализовать. Ведёт через реализацию маленькими шагами (5–15 строк), не исправляет ошибки напрямую — задаёт наводящие вопросы.

### `ds-mentor` — Ментор по Data Science
Проводит через полный DS-пайплайн: понимание задачи → EDA → препроцессинг → выбор метрики → бейзлайн → модель → анализ ошибок. Акцент на принятие решений, а не на заучивание шаблонов.

### `examiner` — Экзаменатор
Принимает зачёт по 5 вопросам нарастающей сложности: пересказ → обоснование → сложность/поведение → граничные случаи → альтернативы. Выставляет оценку X/10 и фиксирует зоны роста.

### `obsidian-exporter` — Экспорт в Obsidian
Автоматически определяет теги, создаёт `[[wiki-links]]` между концепциями и формирует frontmatter. Сохраняет файл в указанный Obsidian vault. Вызывается после `/quiz`.

> **Настройка**: перед первым использованием укажи путь к своему vault в `.claude/agents/obsidian-exporter.md` — найди строку `Сохрани в:` в разделе "Шаг 5: Сохранение" и замени `<obsidian vault path>` на абсолютный путь к папке vault, например `/Users/username/Documents/my_vault`.

## Слэш-команды

| Команда | Что делает |
|---|---|
| `/new-task` | Интерактивно создать задачу перед стартом |
| `/start-task` | Начать разбор задачи из `task.md` |
| `/hint` | Подсказка по уровням (без решения) |
| `/explain <тема>` | Объяснение концепции с поиском через MCP |
| `/review` | Разбор написанного кода |
| `/quiz` | Зачёт → автосохранение заметки → Obsidian |
| `/eda` | EDA для DS-задачи в сократическом режиме |
| `/progress` | Обзор прогресса, слабые места, рекомендации |
| `/export-obsidian` | Ручной экспорт заметки в Obsidian |

### Типичный сценарий работы

```
1. /new-task          → создай задачу интерактивно
   (или вставь условие в .claude/tasks/current/task.md вручную)

2. /start-task        → разбор: понимание → алгоритм → код

3. /hint              → если застрял (уровни 1–4)

4. /review            → когда код написан

5. /quiz              → зачёт + сохранение заметки

6. /export-obsidian   → экспорт в Obsidian vault
```

## Жёсткие правила менторского режима

- Никогда не выдаётся полное решение сразу
- Сначала — объяснение алгоритма словами и псевдокодом
- Код — маленькими частями с объяснением каждой
- После каждой части агент ждёт ответа или вопроса
- Если ты просишь уточнить термин из вопроса — агент объясняет термин, затем **повторяет вопрос и ждёт твоего ответа**

## Использование

### Установка

Клонируй репозиторий в папку, из которой будешь работать с Claude Code:

```bash
git clone https://github.com/arttr1/claude_coding_mentor.git
cd claude_coding_mentor
claude  # открыть Claude Code
```

Либо скопируй папку `.claude/` и файл `CLAUDE.md` в корень своего рабочего проекта.

### Добавление задачи вручную

Открой `.claude/tasks/current/task.md` и заполни шаблон:

```markdown
# Задача

## Условие
Дан массив целых чисел nums и целое число target...

## Источник
LeetCode #1

## Язык / окружение
Python

## Мои первые мысли
Наверное можно перебором...
```

Затем запусти `/start-task`.

## MCP-инструменты для поиска

Команда `/explain` обязательно ищет информацию через MCP перед объяснением. В проекте используются три сервера:

### context7
Получает актуальную документацию библиотек и фреймворков напрямую из источников. Используется когда тема связана с конкретной технологией: pandas, sklearn, PyTorch, LangChain и т.д.

**Настройка** — нужен токен с [context7.com](https://context7.com). Добавь в `~/.claude/claude_desktop_config.json` (или `claude_code_config.json`):
```json
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp@latest"]
    }
  }
}
```

### exa
Семантический поиск в интернете. Находит качественные статьи и объяснения, которых нет в индексируемых поисковиках. Особенно полезен для академических тем и свежих ML-исследований.

**Настройка** — нужен API-ключ с [exa.ai](https://exa.ai):
```json
{
  "mcpServers": {
    "exa": {
      "command": "npx",
      "args": ["-y", "exa-mcp-server"],
      "env": {
        "EXA_API_KEY": "your_key_here"
      }
    }
  }
}
```

### omnisearch
Агрегирует несколько источников: веб-поиск, GitHub, AI-поиск. Используется когда нужно найти примеры кода, репозитории или сравнить подходы.

**Настройка** — нужен API-ключ с [tavily.com](https://tavily.com):
```json
{
  "mcpServers": {
    "omnisearch": {
      "command": "npx",
      "args": ["-y", "@msaelices/mcp-omnisearch"],
      "env": {
        "TAVILY_API_KEY": "your_key_here"
      }
    }
  }
}
```

> Файл конфигурации находится по адресу `~/.claude/claude_code_config.json` (Claude Code CLI) или `~/Library/Application Support/Claude/claude_desktop_config.json` (Claude Desktop).
